# nnican_music_bot

Telegram‑бот для персонального музыкального менеджера (поиск треков, формирование плейлистов и автотегирование).

## Обзор

`nnican_music_bot` разворачивается в контейнерах и использует современный Python‑стек для разработки и продакшена.  
Проект ориентирован на работу через Docker и `make`, чтобы упростить запуск и сопровождение.

## Требования к окружению

Перед запуском убедитесь, что установлены:

-   Python 3.12+
-   Docker
-   docker-compose
-   make
-   uv (менеджер пакетов и окружений для Python)

## Быстрый старт (Docker Compose)

1. Подготовьте файл окружения:

    - Переименуйте `.env.dist` в `.env`.
    - Заполните необходимые переменные (`BOT_TOKEN`, параметры БД и т.п.).

2. Подготовьте конфигурацию Docker Compose:

    - Переименуйте `docker-compose.example.yml` в `docker-compose.yml`.
    - При необходимости отредактируйте порты и настройки сервисов.

3. Соберите образы приложения:

    ```bash
    make app-build
    ```

4. Запустите бота и сопутствующие сервисы:

    ```bash
    make app-run
    ```

5. Для просмотра всех доступных команд используйте:

    ```bash
    make
    ```

## Структура проекта (в общих чертах)

-   `bot/` – код Telegram‑бота (aiogram), хэндлеры и бизнес‑логика.
-   `infra/` или `deploy/` – Docker‑ и compose‑конфигурации.
-   `config/` – файлы конфигурации (env, настройки логирования и т.п.).
-   `Makefile` – цели для сборки, запуска, миграций и утилит.

(Фактическая структура может отличаться; см. комментарии в исходном коде.)
